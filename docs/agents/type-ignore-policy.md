# Type Ignore Policy

This document defines the only acceptable uses of `# type: ignore[...]` in the maintained Python package.

## Core rule

A type suppression is allowed only when the code would otherwise be correct, but the type checker cannot express that correctness precisely enough.

If the suppression does **not** exactly match one of the policy cases below, it must be removed.

## Allowed reasons

Only these two inline reason markers are allowed:

- `# reason: external library has no type support, see docs/agents/type-ignore-policy.md#external-library`
- `# reason: autogenerated code has no type support, see docs/agents/type-ignore-policy.md#autogenerated-code`

No other reason text is allowed.

## External library

Use `# reason: external library has no type support, see docs/agents/type-ignore-policy.md#external-library` **only** when all of the following are true:

1. The error is caused by an upstream library, stub package, plugin, or generated runtime API that does not provide correct or usable type information at that specific call site.
2. The first-party Ralph code is still the right runtime behavior.
3. A small first-party fix is not available. Examples of fixes you must prefer instead:
   - adding or tightening local annotations
   - replacing the ignore with a precise `cast(...)`
   - introducing a typed adapter or wrapper
   - using a supported plugin or typed helper
   - avoiding the dynamic API entirely
4. The suppression is scoped to the narrowest possible error code and line.

If any of those are false, the suppression is **not** justified as `external library` and must be removed.

### External library is NOT allowed for

These are not valid justifications for `external library`:

- convenience
- speed of implementation
- uncertainty about the right fix
- broad model-level suppressions that can be solved in first-party code
- test-only shortcuts when the test can be typed correctly
- places where the code is actually wrong and the ignore hides that fact

## Autogenerated code

Use `# reason: autogenerated code has no type support, see docs/agents/type-ignore-policy.md#autogenerated-code` only when the ignored line is mechanically produced or tightly constrained by generated output and a manual local typing fix is not practical.

Hand-written code must not claim `autogenerated code`.

## Review standard

Reviewers should ask one question first:

> Is the type checker blocked by the external system, or by our own code?

If the blocker is our own code, remove the suppression and fix the typing.
If the blocker is the external system, keep the suppression narrow and use the exact external-library policy suffix.

## Required inline format

All suppressions must use a specific mypy error code:

```python
value = external_api()  # type: ignore[assignment]  # reason: external library has no type support, see docs/agents/type-ignore-policy.md#external-library
```

Blanket `# type: ignore` comments are forbidden.
