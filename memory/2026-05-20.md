# 2026-05-20

- Important marketing positioning saved: **"The operating system for autonomous coding."** should be used as a core Ralph Workflow slogan / positioning line. Older “Repo-Native” framing should be de-emphasized or replaced with this autonomous-coding positioning.
- Standing positioning preference: do not use “Repo-Native” framing for Ralph Workflow. Replace it with stronger autonomous-coding positioning, especially **“The operating system for autonomous coding.”**
- Stronger standing loop rule: every self-improvement loop must require third-party verification at every claimed improvement state. No self-improvement loop may self-certify success. If a verifier fails to sign off, the loop must automatically trigger another remediation pass and then a fresh independent verifier.
- Docs-watchdog repair lesson: a loop is not fixed if any relevant surface is outside the watched/repairable scope. The Ralph docs watchdog had a real scope hole because the GitHub mirror README was not initially part of the guarded surface.
- Durable docs-watchdog structure now includes: deterministic checker, deterministic runner, separate verifier, separate runner/verifier artifacts, scheduled runner, scheduled verifier, loop registry, and recurring loop-integrity audit.
- Ralph docs-watchdog scope now explicitly includes the primary repo README, crate README, promoted docs target checks, and the GitHub mirror README so README link-farm regressions cannot hide in the mirror copy.
- Process failure lesson: do not treat policy text, one good run, or partial surface cleanup as equivalent to full enforcement. Closure requires executable enforcement, scheduled execution, durable artifacts, and independent verifier signoff on the actual final state.
- New durable docs-watchdog standard: treat the top-level docs system holistically, not as isolated README fixes. The enforced review surface must include README, START_HERE, and docs index together as one user journey, plus adjacent promoted next-click pages when they shape first-run understanding.
- New durable docs quality rule: all major criteria matter simultaneously. Do not trade away marketability, copyedit quality, easy-to-understand language, 10-second clarity, information hierarchy, low link anxiety, developer fit, non-developer expectation-setting, and cross-surface coherence just because a narrower structural checker passes.
- New durable marketing/docs rule: the top-level README path should work for both developers and non-developers by being easy to understand quickly, while also making it obvious who Ralph Workflow is *not* for. Non-developers or readers who cannot judge a reviewable software handoff should be able to self-disqualify early instead of being misled by top-level copy.
- New durable enforcement lesson: runner, checker, verifier, and integrity audit must agree on the same canonical docs-system standard. If any component carries stale templates or brittle one-line expectations, the loop can silently reintroduce bad copy or fight its own cleanup.
