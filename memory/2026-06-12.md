## Root cause (architectural, not behavioral)
The D50 BINDING in `apollo_marketer_prompt.md` (line 40) literally said "MUST fire H2 via the headed-browser computer-use path" — a public-write directive that contradicted D46 PUBLIC-WRITE CONDUCT ("the owner posts, never you") at the verb level. The 3 gh-write-guard blocks today were the loop trying to satisfy this self-contradicting binding. The shim is correct to refuse; the prompt was wrong to bind.

The 2 of 3 read-intent blocks (graphql + search/repos) happened because the prompt's BINDING implied a discovery step ("log in to HN by hand using the mouse/keyboard") but the prompt was silent on which `gh` subcommands are safe for the discovery path. The loop invented `gh api` calls; the shim blocked them. (The shim blocks ALL `gh api` because the verb is opaque at parse time — `gh api` is one command; the verb lives in the body or `-X` flag.)

## Fix shipped (this run)
1. **D50 BINDING → D70 BINDING** in the prompt: the loop's job for H2 is now (a) keep the canonical paste-ready packet at `drafts/2026-06-10_launch_assets_READY.md` §H2 fresh, (b) maintain the one-paste HTML mirror at `drafts/2026-06-10_H2_HN_PASTE_READY.md` for a 90-second owner paste, (c) log `tactic="pending_owner_approval"` + append a queue pointer every run that observes a Tue/Wed/Thu 14-16 UTC window. DRAFT verbs only — no fire/submit/post/open/create on public surfaces.
2. **GH-READ SAFE-LIST** (D70 supplement) added to the prompt: a catalogue of safe `gh` read subcommands (the verb-in-command-name form, which the shim does NOT block) + a small allowlist of flag-less `gh api` reads (`gh api users/<login>`, `gh api repos/<owner>/<repo>`, `gh api repos/<owner>/<repo>/contents/<path>`, `gh api repos/<owner>/<repo>/issues/<n>`, `gh api repos/<owner>/<repo>/issues/<n>/comments`, `gh api repos/<owner>/<repo>/releases`, `gh api repos/<owner>/<repo>/events`). The supplement explicitly BANS `gh api graphql` (requires `-f`), `gh api -f/--field/--raw-field/--input` (defaults POST), `gh api search/*` (use `gh search ...` instead), and `gh api -X POST|PUT|PATCH|DELETE` (the shim's method-flag regex catches all of these).
3. **PRE-MISSION SELF-CHECK** at the top of the prompt: every run's first turn-stanza MUST (a) state the discovery goal, (b) list the `gh` calls planned with safe-list line numbers, (c) REWRITE or DROP any call not in the safe list, (d) for any read not in the safe list, DRAFT a `gh api <url>` call to `drafts/<date>_<gh_read>_PENDING_OWNER.md` for the owner to run personally + log as `tactic="pending_owner_approval"`. This is the tripwire that catches the next `gh api` block before the shim does.
4. **`public_write_binding` rule added to `apollo_prompt_lint.py`** + selftest fixture: catches a BINDING block that uses fire/submit/post/open/launch/create/publish/send + a public-surface target (HN/Reddit/dev.to/Mastodon/X/LinkedIn/GitHub issue or PR) within a tight 90-char/60-char window. Exemptions: whole-unit `BINDING:` or `DISCIPLINE:` catalogue (the D70 BINDING itself is allowlisted as a catalogue-of-prohibitions), and in-line `**POST / fire / open / create / submit**` bold-wrapped banned-verb catalogues.
