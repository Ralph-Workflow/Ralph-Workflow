### 2️⃣ Agents CLI (Phoenix Labs) — Meta-harness for CLI agents
- **Source:** Show HN, HN #48346958 (May 31 — very fresh)
- **Score:** 6 pts, launched extremely recently
- **What:** Open-source meta-harness: shims for claude/codex, auto-rotation of CC credentials to handle session limits, multi-layer dot-agents repos, shared project resources (skills, plugins, rules).

### 3️⃣ Haystack — PR triage for agent-generated code
- **Source:** Show HN, HN #48182856 (May 18)
- **Score:** 45 pts, 17 comments
- **What:** Routes AI-agent PRs into 3 buckets: safe-to-merge, needs-fixes, needs-human-review. Uses diffs + agent conversation to classify.
- **Signal:** "Each member of our team can pump out 20+ PRs a day — code review became cognitively exhausting"
- **Ralph connection:** Directly validates Ralph's "would you merge it?" positioning. Ralph's bounded-task design produces reviewable finish states — Haystack is downstream tooling for exactly that output.
- **Action:** Blog post angle: "Your Agent Generated 20 PRs Today — Now What?" Connect Haystack + Ralph as complementary: Ralph produces clean reviewable work, Haystack helps you scale the review itself.

### 4️⃣ Mainline — Intent recording for agent commits
- **Source:** Show HN, HN #48247718 (May 23)
- **Score:** 4 pts
- **What:** CLI tool that hooks into coding agents to store intent/decisions/risks in git before changes. Review intent before reviewing code.
- **Signal:** "Agents fall into the same pitfalls — issues not discovered until merge phase"
- **Ralph connection:** Directly maps to Ralph's planning loop — Ralph already surfaces spec and intent before building.
- **Action:** Evaluate whether Ralph's planning loop output could feed Mainline hooks. Positioning: "Ralph Workflow doesn't just record intent — it enforces it through bounded phases."

### 5️⃣ Twill.ai — Cloud sandbox delegation
