# JSON Feed feed_url Spec Compliance Fix
- **Date**: 2026-05-30T05:41 UTC
- **Commit**: `7a431af`
- **Release**: `20260530054141`

## Problem
JSON Feed at `https://ralphworkflow.com/blog/feed.json` had `feed_url` set to `https://ralphworkflow.com/blog/feed` — the RSS endpoint path. Per JSON Feed 1.1 spec, `feed_url` MUST be the URL of the feed itself so feed readers and AI crawlers can uniquely identify and refresh it.

## Root Cause
Both RSS and JSON feeds share the same route path `/blog/feed` with format constraints. The `blog_feed_json_url` route helper resolves to `/blog/feed` without the `.json` extension.

## Fix
Appended `.json` to the route helper in `feed_json.json.jbuilder`:
```ruby
json.feed_url "#{blog_feed_json_url}.json"
```
