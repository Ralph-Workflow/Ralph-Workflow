## Core Research Findings

### 1. RalphWorkflow's strongest SEO angle is not generic "AI coding"
Trying to rank for broad terms like "AI coding tools" will be hard.

The best opening is the **workflow/orchestration/spec-driven** layer:
- unattended runs
- repeatable dev/review loops
- CLI orchestration
- spec-first agent workflows

That is much more ownable than generic assistant comparisons.

### 2. The site is in better technical shape than the older reports suggested
Some older notes are now stale.

Today, the live site clearly has:
- canonical
- JSON-LD
- OG/Twitter metadata
- robots.txt
- sitemap.xml

So the main bottleneck has shifted from metadata cleanup to **content depth + search capture**.

### 3. RalphWorkflow needs search-entry pages, not just a solid homepage
The homepage is decent, but SEO growth will come from pages that match specific search intent.

Best candidates:
- "How to run Claude Code unattended"
- "Spec-driven AI agent workflow"
- "AI agent orchestration CLI"
- "Unattended coding agent workflow"
- comparison pages / roundup participation

### 4. The title can still be sharpened
Current title:
- `AI agent workflow composer — Ralph Workflow`

It is solid, but probably underuses search terms people actually type.
Potential improvement direction:
- include **CLI**
- include **unattended** or **automation**
- keep the brand name

Example direction:
- `Ralph Workflow — AI Agent Orchestration CLI for Unattended Coding Workflows`
