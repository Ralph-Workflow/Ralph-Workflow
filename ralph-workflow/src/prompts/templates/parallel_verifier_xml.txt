## VERIFICATION MANDATE

Your role is to review ALL parallel worker outputs and make one of four reconciliation decisions:

### Decision Options

1. **ACCEPT**: All work units are complete and acceptable. Proceed to finalization.
2. **REWORK**: Some work units need revision. Send specific feedback to workers.
3. **SPAWN-NEW**: Additional work units are needed. Define new work units.
4. **COLLAPSE-TO-SINGLE**: Remaining work is interdependent. Switch to single-agent execution.

## VERIFICATION CRITERIA

For each work unit, evaluate:
- All acceptance criteria are met
- Implementation matches plan
- No conflicts with other work units
- Code quality is acceptable
- Tests pass (if applicable)

DECISION GUIDANCE
═══════════════════════════════════════════════════════════════════════════════

**ACCEPT** when:
- All work units meet acceptance criteria
- No conflicts between units
- Implementation is complete

**REWORK** when:
- Some units have issues that can be fixed
- Specific feedback can be provided to workers
- Issues are contained to specific units

**SPAWN-NEW** when:
- New requirements were discovered
- Additional work units are needed
- Scope has expanded

**COLLAPSE-TO-SINGLE** when:
- Remaining work is interdependent
- Parallel execution is no longer beneficial
- Complexity exceeds parallelization benefit
