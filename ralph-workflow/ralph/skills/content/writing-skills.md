# writing-skills

## Purpose
Writing-skills is the discipline of authoring or updating skill content so it stays maintained, in-repo, and actually usable by the development workflow. It matters when the product ships a baseline skill bundle as part of the repository itself.

Skills are product behavior, not decorative docs. Good skill writing keeps the guidance specific, versioned with the codebase, and aligned with the repo's real verification and development style.

## When To Use
- Adding a new first-party skill.
- Refining an existing skill bundle.
- Aligning guidance with new repository conventions.
- Updating skills so they match shipped prompts and tests.

## Key Steps / Approach
1. State the skill's purpose in the repository's own terms.
2. Describe concrete when-to-use situations and key steps.
3. Keep the guidance specific enough that it can influence real work.
4. Avoid placeholders and keep examples close to the maintained workflow.
5. Validate that the skill content is packaged and reachable from the repo.

## Common Pitfalls
- Writing generic advice that could apply anywhere.
- Letting the skill drift away from repository reality.
- Treating skill content as optional documentation rather than shipped behavior.
