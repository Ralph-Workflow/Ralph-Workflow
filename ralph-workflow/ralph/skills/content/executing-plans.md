# executing-plans

## Purpose
Executing-plans is the skill for converting an approved written plan into concrete repo changes. It keeps you from re-litigating the scope once the plan is already good enough to run, and it emphasizes disciplined follow-through over exploratory digression.

This skill helps preserve momentum and accountability. It also keeps execution aligned with the plan's proof requirements so the final artifact can be justified without hand-waving.

## When To Use
- A plan has already been approved.
- You are ready to modify files and run checks.
- The remaining work is implementation rather than discovery.
- You need to keep each step tied to its proof and verification.

## Key Steps / Approach
1. Work the plan in order unless a documented parallel path is safe.
2. Make the smallest change that satisfies the current step.
3. Verify each logical unit before marking it done.
4. Keep proof explicit: changed files, commands run, and resulting state.
5. Do not reopen scope unless the repository reality forces a correction.

## Common Pitfalls
- Turning execution into a second planning pass.
- Batching verification until the very end.
- Letting scope creep spill beyond the approved plan.
