# Claude Code Automation for Real Repo Work

If you are searching for **Claude Code automation**, the real question is not just how to keep Claude Code running while you are away.

The real question is whether you come back to a **reviewable result** instead of a long session and a confident done claim.

Ralph Workflow is a **free and open-source** orchestration CLI that runs the coding agents you already use **on your own machine**.

It is for developers and technical teams with work that is **too big to babysit and too risky to trust blindly**.

## What useful Claude Code automation should solve

Useful Claude Code automation should make the **finish state** clearer, not just the runtime longer.

It should leave you able to answer:

- what changed
- what checks ran
- what still needs human judgment
- whether you would merge it

That is the gap Ralph Workflow is meant to close.

## What Ralph Workflow adds on top of Claude Code

Ralph Workflow does **not** replace Claude Code.

It wraps the agent you already use in a repo-native unattended flow so the result is easier to judge in normal engineering terms.

## Fastest honest first run

1. Keep Claude Code already installed and authenticated on your own machine
2. Pick one real backlog task with clear acceptance criteria
3. Run Ralph Workflow overnight
4. Review the diff, checks, and artifacts in the morning
5. Ask: **would I merge this?**

If you want the shortest path, start with [Try Ralph Workflow on one real backlog task](../START_HERE.md).
