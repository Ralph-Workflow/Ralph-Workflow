# Type-Ignore Policy

This document defines the policy that `ralph.testing.audit_typecheck_bypass`
enforces. The two accepted reason markers are anchors in this file so that
contributors and reviewers can click straight to the rationale.

## When a `# type: ignore` is acceptable

`# type: ignore[CODE]` is acceptable **only** when both conditions hold:

1. The suppression is for a third-party library, autogenerated code, or other
   out-of-tree code that genuinely cannot be typed by Ralph.
2. The suppression is annotated with one of the reason markers below on the
   same logical line.

## `# reason: external library` {#external-library}

A `# type: ignore[CODE]` may be annotated with:

```python
# reason: external library has no type support,
# see docs/agents/type-ignore-policy.md#external-library
```

when the suppression covers a third-party library call (e.g. an
`httpx`, `sentry-sdk`, `mcp`, `gitpython`, or `rich` symbol whose upstream
type stub is wrong, missing, or partial). The marker must be on the same
logical line as the suppression.

## `# reason: autogenerated code` {#autogenerated-code}

A `# type: ignore[CODE]` may be annotated with:

```python
# reason: autogenerated code has no type support,
# see docs/agents/type-ignore-policy.md#autogenerated-code
```

when the suppression covers code emitted by a code generator (e.g. a parser
table, a command registry, a display ring buffer, or a manifest built by
Hatch). The marker must be on the same logical line as the suppression.

## What is NOT acceptable

- Bare `# type: ignore` without a code: the project enables
  `ignore-without-code` in `mypy.ini` and rejects them.
- `# type: ignore[CODE]` without one of the two reason markers: rejected.
- `# type: ignore` inside test files: rejected. Tests must be fully typed.
- A blanket `# type: ignore[explicit-any]` on first-party Pydantic models:
  rejected. Use the Pydantic compatibility shim.
- `# pyright: ...` runtime directives in non-test code: rejected. Use a
  `# type: ignore[CODE]` with a reason marker.

## How the audit enforces this

`ralph.testing.audit_typecheck_bypass` walks the repository and verifies that
every `# type: ignore[CODE]` line either ends with one of the two reason
markers, or is in a path the audit exempts (e.g. the audit's own source). Test
files are rejected unconditionally. The audit is part of `make verify`.

## Adding a new acceptable reason

Do not. If a new category of suppression becomes necessary, open a PR that
adds an anchor to this document and the corresponding entry to
`_VALID_REASON_MARKERS` in `ralph/testing/audit_typecheck_bypass.py`. Both
must change in the same commit.
