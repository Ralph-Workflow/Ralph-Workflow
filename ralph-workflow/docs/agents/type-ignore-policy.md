# Type-Ignore Policy

The `mypy --strict` run for this repository forbids any blanket
`# type: ignore` comment. Every suppression MUST include a specific
error code (`# type: ignore[<code>]`) and end with one of the two
reason suffixes below.

The reason suffix is the single point of accountability: it documents
why the suppression is acceptable, and a code review can grep for
unannotated suppressions.

## External library

A `# type: ignore[CODE]` comment may be used when the suppression is
necessary because an external library lacks the type annotations mypy
needs. Every such suppression MUST end with:

```
# reason: external library has no type support, see docs/agents/type-ignore-policy.md#external-library
```

## Autogenerated code

A `# type: ignore[CODE]` comment may be used when the suppression is
necessary because a code generator emits code mypy cannot reason
about (e.g. dynamically constructed dataclasses, factories). Every
such suppression MUST end with:

```
# reason: autogenerated code has no type support, see docs/agents/type-ignore-policy.md#autogenerated-code
```

## What is NOT acceptable

- Blanket `# type: ignore` without a specific error code.
- `# type: ignore[CODE]` without one of the two reason suffixes.
- `# type: ignore[CODE] # reason: ...` that does not match one of the
  two allowed suffixes verbatim.
- Any suppression in a test file. Tests must be fully typed.
