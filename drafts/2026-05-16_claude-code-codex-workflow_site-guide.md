# Claude Code + Codex Workflow: Plan, Build, Review

## The problem
AI coding tools often say they are done before the work is actually solid. The real gap is not just code generation. It is getting back something you would actually review and merge.

## A workflow that works better
1. Sharpen the task before code starts
2. Run one scoped task in isolation
3. Build and verify in the same loop
4. End with a reviewable diff, checks, and open questions

## Why people use Claude Code and Codex together
One tool can implement. Another can review, challenge, or verify. That is already better than one long unchecked run. But the manual glue work still becomes the bottleneck.

## Where Ralph Workflow fits
Ralph Workflow is built for the gap between "the agent says done" and "the result actually holds up."

It helps turn a delegated coding task into:
- a sharpened request
- a build + verify loop
- a finished diff
- a reasoning trail
- a clean re-entry point

## The test
Ask one question at the end:

**Would you merge this?**

If yes, the workflow worked.
If not, the workflow still needs improvement.
