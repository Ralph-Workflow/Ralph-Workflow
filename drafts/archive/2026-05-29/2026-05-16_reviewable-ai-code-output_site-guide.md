# Reviewable AI Code Output

## Why this matters
AI coding output is only useful if a human can review it quickly and decide whether it actually holds up.

A lot of tools optimize for generation speed.
Ralph Workflow optimizes for the finish: a result you can inspect, test, and decide to merge.

## What reviewable output looks like
Reviewable AI code output means:
- the task scope is clear
- the changed files make sense
- the diff is traceable
- checks actually ran
- uncertainty is called out instead of hidden

## What bad output looks like
Bad output usually has one or more of these problems:
- the scope drifted
- the diff is too broad
- the summary is more confident than the result
- the checks are unclear or missing
- the human has to reconstruct what happened from scratch

## Better standard
The standard should be simple:
- sharpen
- build
- verify
- review

If the result cannot survive that sequence, it is not done.

## Where Ralph Workflow fits
Ralph Workflow helps turn AI coding into a reviewable result instead of a chat transcript plus a pile of changed files.

That is the real difference between output and adoption-worthy workflow.
