I wrote this because most AI coding discussion still focuses on model quality, but the workflow gap feels more important: agents say "done" before the result is something you'd actually merge.

The product angle behind the piece is Ralph Workflow: a free and open-source tool that orchestrates the coding agents you already use on your own machine, so longer unattended runs end in something reviewable instead of just a transcript. Repo if you want to inspect the implementation: https://codeberg.org/RalphWorkflow/Ralph-Workflow (primary). Likely reply angles:
- worktrees solve file conflicts, not semantic trust
- code review is still the bar; the workflow should hand you a reviewable diff instead of forcing you to reconstruct the run
- unattended coding is only worth it when the morning-after result is small enough to inspect honestly
- the point is not automation theater; it is proof the task holds up
